Changes from 0.3.3-dev to 0.3.4-dev

Warning:
- This development build has not been tested in-game yet. Please test it on a backup/test server before using it on a live server.

- Added the first Aethelguard diagnostics system for admins.
- Added /aethelguard diagnostics for a quick plugin health overview.
- Added /aethelguard diagnostics player <player> for account/auth state inspection.
- Added /aethelguard diagnostics config for common config warnings.
- Added /aethelguard diagnostics dump to write a support-friendly diagnostics text file.
- Added diagnostics config settings:
  - diagnostics.enabled
  - diagnostics.dump.enabled
  - diagnostics.dump.folder
  - diagnostics.mask-sensitive-data
  - diagnostics.include-player-ip
  - diagnostics.max-dumps-to-keep
- Diagnostics overview now reports:
  - plugin version
  - storage mode
  - database health
  - language settings
  - PIN/PIN GUI state
  - captcha state
  - two-factor state
  - recovery state
  - adaptive security and VPN check state
  - session state
  - local users folder and user index state
  - active auth session count
  - waiting auth/captcha/2FA counts
- Player diagnostics now reports:
  - registration/authentication state
  - auth mode
  - password/PIN availability
  - two-factor state
  - captcha requirement
  - wrong password/PIN attempts
  - recovery method
  - security question and backup code status
  - last login/world/location
  - last IP, hidden or masked according to config
- Config diagnostics now warns about:
  - invalid PIN GUI themes
  - unsafe database table names
  - unsafe local users folder names
  - invalid sound names
  - invalid diagnostics dump retention values
- Diagnostic dumps intentionally avoid passwords, PIN hashes, 2FA secrets, backup codes, and other private secrets.
- Diagnostic dump retention cleanup was added so old dump files are removed after the configured limit.
- Added tab completion for the diagnostics command family.
- Added English and Turkish message keys for diagnostics output.
- Added diagnostics keys to the automatic config sync/default system.
