Changes from 0.3.5-dev to 0.3.6-dev

Warning:
- This development build has not been tested in-game yet. Please test it on a backup/test server before using it on a live server.

- Added temporary auth lockout for repeated failed authentication cycles.
- The lockout cycle threshold is intentionally hardcoded to 2.
- The first full max-attempt failure cycle still uses the normal wrong password/PIN kick.
- If the same player reaches the max-attempt kick cycle again, Aethelguard applies a temporary auth lock and kicks the player with a lockout message.
- Players with an active temporary auth lock are kicked immediately when they try to join again.
- The temporary lockout duration is intentionally hardcoded to 5 minutes.
- Added a simple config switch:
  - auth-settings.temporary-lockout.enabled
- Added English and Turkish lockout kick messages:
  - messages.auth-lockout-triggered-kick
  - messages.auth-lockout-active-kick
- Successful authentication now clears wrong password attempts, wrong PIN attempts, temporary lockout state, and failed auth kick cycle tracking.
- Temporary auth lockout currently applies to password and PIN max-attempt kick cycles.
- Captcha attempt handling remains separate from temporary auth lockout.
